# 第一阶段五人并行开发任务书

## 1. 文档目的

本文档用于指导五名成员并行完成 UAV-USV 协同追踪系统的第一阶段。

除项目负责人外，其余四名成员都按 ROS 2 初学者考虑。因此第一阶段不追求复杂算法，
而是先建立清楚、稳定、可以测试的数据链路。每个人只需要完成自己负责的小模块，
最终由项目负责人将模块组合起来。

第一阶段建议周期为 **两周（10 个工作日）**。

## 2. 第一阶段最终效果

第一阶段结束时，系统应完成下面这条链路：

```text
Gazebo 中的目标船按照固定路线移动
                 ↓
        发布目标船真实位置
          ↙               ↘
无人机模拟视觉观测       船体模拟雷达观测
          ↘               ↙
        项目负责人进行目标融合
                 ↓
       选择需要追踪的目标船
          ↙               ↘
  向无人机发布目标点      向无人船发布目标点
          ↓               ↓
  无人机执行追踪       无人船使用 Nav2 追踪
                 ↓
       RViz 显示完整追踪过程
```

组网模块同时完成：

```text
uav_01、uav_02、usv_01、usv_02
              ↓
       发布设备心跳和状态
              ↓
      生成当前在线设备列表
```

第一阶段验收时至少满足：

1. 一个目标船可以在 Gazebo 中连续运动。
2. 无人机侧和无人船侧都能发布标准目标观测。
3. 融合节点可以发布一个统一目标。
4. 无人船可以持续向目标移动。
5. 无人机可以接收追踪目标点。
6. 组网节点可以显示四个模拟成员在线或离线。
7. 所有目标、载体、TF 和目标点可以在 RViz 中查看。
8. 所有包可以通过工作区检查脚本。

## 3. 第一阶段暂时不做什么

为了避免新成员一开始被复杂问题卡住，本阶段暂时不要求：

- 真实 YOLO 船舶识别。
- 真实 MID-360 驱动。
- FAST-LIVO 或其他 SLAM。
- 多架真实 PX4 SITL 同时飞行。
- 复杂 EKF、多目标数据关联或深度学习融合。
- 真实无线自组网。
- AIS、DCPA、TCPA 和 COLREGs。
- 最优围捕、队形控制或强化学习。

这些功能以后用真实节点替换第一阶段的模拟节点，第一阶段接口不需要推倒重来。

现有 AIS 和 COLREGs 文件保留，但本阶段不继续开发：

```text
src/uav_usv_colregs
src/uav_usv_sim/scripts/ais_simulator.py
src/uav_usv_sim/scripts/colregs_scenario_controller.py
src/uav_usv_sim/config/colregs_scenarios.yaml
```

## 5. 公共接口约定

公共接口由项目负责人管理。其他成员需要新增或修改公共消息时，先在 Issue 或群里说明，
由项目负责人统一修改 `uav_usv_interfaces`。

### 5.1 坐标与单位

- 全局坐标系统一使用 `map`。
- ROS 坐标采用 ENU。
- 距离单位使用米。
- 速度单位使用米每秒。
- 角度单位使用弧度。
- Z 轴向上。
- 时间戳使用仿真消息产生时刻。

### 5.2 第一阶段 Topic

| Topic | 类型 | 发布者 | 订阅者 |
|---|---|---|---|
| `/simulation/ground_truth/targets` | `TrackedObjectArray` | 成员 A | 成员 B、成员 C、项目负责人 |
| `/uav_01/perception/tracks` | `TrackedObjectArray` | 成员 B | 项目负责人 |
| `/usv_01/perception/tracks` | `TrackedObjectArray` | 成员 C | 项目负责人 |
| `/perception/tracks/fused` | `TrackedObjectArray` | 项目负责人 | 任务节点、RViz |
| `/mission/selected_target` | `TrackedObject` | 项目负责人 | 任务节点、RViz |
| `/uav_01/goal_pose` | `PoseStamped` | 项目负责人 | 成员 B |
| `/usv_01/goal_pose` | `PoseStamped` | 项目负责人 | Nav2/成员 C |
| `/fleet/heartbeats` | `FleetMember` | 各设备模拟节点 | 成员 D |
| `/fleet/members` | `FleetMemberArray` | 成员 D | 项目负责人、RViz |

消息包全名为：

```text
uav_usv_interfaces/msg/TrackedObject
uav_usv_interfaces/msg/TrackedObjectArray
geometry_msgs/msg/PoseStamped
```

### 5.3 `TrackedObject` 填写规则

第一阶段所有目标消息必须满足：

```text
header.frame_id = "map"
classification = CLASS_VESSEL
dimensions.x > 0
dimensions.y > 0
confidence 在 0.0 到 1.0 之间
```

来源字段：

```text
Gazebo真值：SOURCE_UNKNOWN
无人机观测：SOURCE_CAMERA
船体观测：SOURCE_LIDAR
融合结果：SOURCE_FUSED，同时保留 CAMERA 和 LIDAR 位
```

同一艘目标船的 `track_id` 暂时统一为：

```text
target_01
```

### 5.4 组网消息

第一阶段需要由项目负责人先在 `uav_usv_interfaces/msg` 中增加：

```text
FleetMember.msg
FleetMemberArray.msg
```

建议 `FleetMember.msg` 内容：

```text
uint8 TYPE_UNKNOWN=0
uint8 TYPE_UAV=1
uint8 TYPE_USV=2

std_msgs/Header header
string vehicle_id
uint8 vehicle_type
bool online
geometry_msgs/Pose pose
geometry_msgs/Twist twist
float32 battery_percentage
string current_task
```

建议 `FleetMemberArray.msg` 内容：

```text
std_msgs/Header header
FleetMember[] members
```

只有项目负责人负责最终确定并提交消息定义，避免五个人同时修改接口包。

#### 任务 2：编写简单融合节点

建议文件：

```text
src/uav_usv_perception/scripts/target_fusion_node.py
```

节点订阅：

```text
/uav_01/perception/tracks
/usv_01/perception/tracks
```

节点发布：

```text
/perception/tracks/fused
```

第一阶段不需要复杂 EKF，可以使用简单方法：

1. 按 `track_id` 匹配目标。
2. 检查两条观测时间差不超过 0.5 秒。
3. 使用置信度进行加权平均。
4. 若只有一个来源，则直接发布该来源结果。
5. 超过 2 秒没有更新的目标从列表删除。

位置融合示意：

```text
fused_x = (camera_x * camera_confidence + lidar_x * lidar_confidence)
          / (camera_confidence + lidar_confidence)
```

#### 任务 3：编写追踪任务节点

建议文件：

```text

```

节点订阅：

```text

/fleet/members
```

节点发布：

```text
/mission/selected_target
/uav_01/goal_pose
/usv_01/goal_pose
```

第一阶段目标点规则可以简单设置为：

- 无人机目标点：目标船后方 5 米、上方 8 米。
- 无人船目标点：目标船后方 10 米。
- 目标点每秒更新 2 到 5 次。
- 目标丢失超过 2 秒后停止更新目标点并报警。

## 14. 遇到问题时怎么处理

### 编译失败

先只编译自己的包，并查看第一个错误：

```bash
colcon build --packages-select 包名 --symlink-install \
  --event-handlers console_direct+
```

### Topic没有数据

```bash
ros2 topic list
ros2 topic info /topic名称
ros2 topic hz /topic名称
ros2 topic echo /topic名称
```

### TF错误

```bash
ros2 run tf2_ros tf2_echo 父坐标系 子坐标系
ros2 run tf2_tools view_frames
```

### 不知道是否应该修改别人的文件

不要直接修改。先在 Issue、Pull Request 或团队群中说明：

```text
我需要什么输入
现在的接口为什么不够
我建议怎样修改
会影响哪些模块
```

由项目负责人决定接口是否修改。
