# UAV_USV 协同仿真复现运行指南

这份文档用于说明别人拿到 `UAV_USV` 包后，需要准备哪些环境、如何编译、如何启动 PX4 + Gazebo + 协同任务，以及常用参数是什么意思。

## 1. 基本环境

当前工程按下面环境开发和测试：

- Ubuntu 22.04
- ROS 2 Humble
- Gazebo Sim 8
- PX4-Autopilot，路径请按自己电脑上的实际位置设置
- Python 3
- `pymavlink`
- Gazebo Python Transport 相关绑定，例如 `gz.transport13`、`gz.msgs10`

ROS 依赖主要来自 `src/uav_usv_sim/package.xml`：

- `ament_cmake`
- `ament_index_python`
- `launch`
- `launch_ros`
- `rclpy`
- `std_srvs`
- `python3-pymavlink`

Gazebo 插件编译依赖主要来自 `CMakeLists.txt`：

- `gz-sim8`
- `gz-plugin2`
- `gz-math7`
- `gz-transport13`
- `gz-msgs10`

如果缺少 ROS 依赖，可以先尝试：

```bash
sudo apt update
sudo apt install python3-pymavlink
```

如果 Gazebo 相关 `pkg-config` 包缺失，需要安装 Gazebo Sim 8 对应开发包。

## 2. 目录约定

下面路径只是示例，使用时请改成自己电脑上的实际路径：

```bash
export UAV_USV_WS=/你的路径/UAV_USV
export PX4_DIR=/你的路径/PX4-Autopilot
export UAV_USV_INSTALL=/var/tmp/UAV_USV_install
export UAV_USV_BUILD=/var/tmp/UAV_USV_build
export UAV_USV_LOG=/var/tmp/UAV_USV_log
```

其中：

- `UAV_USV_WS` 是本 ROS 2 工作区。
- `PX4_DIR` 是 PX4 源码目录。
- `UAV_USV_INSTALL` 是编译后的 install 目录。
- `UAV_USV_BUILD` 和 `UAV_USV_LOG` 是构建缓存和日志目录。

建议每次启动时显式传入 `px4_dir`，或者提前 `export PX4_DIR=...`：

```bash
ros2 launch uav_usv_sim uav_usv_px4_sim.launch.py px4_dir:=$PX4_DIR
```

## 3. 编译工作区

由于 `/home` 空间可能比较紧张，推荐把 `build`、`install`、`log` 放到 `/var/tmp`：

```bash
export UAV_USV_WS=/你的路径/UAV_USV
export UAV_USV_INSTALL=/var/tmp/UAV_USV_install
export UAV_USV_BUILD=/var/tmp/UAV_USV_build
export UAV_USV_LOG=/var/tmp/UAV_USV_log

cd $UAV_USV_WS
source /opt/ros/humble/setup.bash

colcon --log-base $UAV_USV_LOG build \
  --build-base $UAV_USV_BUILD \
  --install-base $UAV_USV_INSTALL \
  --symlink-install
```

之后每次运行前都要 source：

```bash
source /opt/ros/humble/setup.bash
source $UAV_USV_INSTALL/setup.bash
```

如果你改了 C++ 插件或 Python 脚本，重新执行上面的 `colcon` 命令即可。

## 4. 仿真资源同步到 PX4

PX4 启动 Gazebo 时，会读取 PX4 自己目录下的 world 文件和 Gazebo 模型路径。本包提供了同步脚本：

```bash
source /opt/ros/humble/setup.bash
source $UAV_USV_INSTALL/setup.bash

ros2 run uav_usv_sim sync_to_px4.sh
```

这个脚本会做几件事：

- 把 `worlds/default.sdf` 同步到 PX4 的 Gazebo worlds 目录。
- 把 `simple_boat` 和 `waves` 模型同步到 `~/.gz/models`。
- 把本包编译出的 Gazebo 插件路径写入 world/model 文件。
- 确保 PX4 启动时能加载无人船、海浪、灯塔和停机坪约束插件。

正常使用 `uav_usv_px4_sim.launch.py` 或 `uav_usv_cooperation_demo.launch.py` 时，launch 会自动先执行同步脚本，一般不需要手动运行。

## 5. 单独启动 Gazebo 世界

如果只想看世界、船、灯塔和海面，不启动 PX4：

```bash
source /opt/ros/humble/setup.bash
source $UAV_USV_INSTALL/setup.bash

ros2 run uav_usv_sim run_gz_world.sh
```

如果想带键盘控制船：

```bash
source /opt/ros/humble/setup.bash
source $UAV_USV_INSTALL/setup.bash

ros2 launch uav_usv_sim uav_usv_world_keyboard.launch.py
```

## 6. 分步启动 PX4 协同任务

第一步，启动 PX4 + Gazebo。无人机会出生在无人船后方停机坪上：

```bash
source /opt/ros/humble/setup.bash
source $UAV_USV_INSTALL/setup.bash

ros2 launch uav_usv_sim uav_usv_px4_sim.launch.py px4_dir:=$PX4_DIR
```

第二步，另开一个终端，启动协同任务节点：

```bash
source /opt/ros/humble/setup.bash
source $UAV_USV_INSTALL/setup.bash

ros2 launch uav_usv_sim cooperative_lighthouse_mission.launch.py
```

任务流程：

1. 节点通过 MAVLink 连接 PX4。
2. 无人机切入 OFFBOARD。
3. 停机坪约束插件释放无人机。
4. 无人机平滑起飞到指定高度。
5. 无人船先向灯塔移动。
6. 无人机延迟一段时间后飞向灯塔周围的安全半径点，而不是直接撞向灯塔中心。
7. 船到达灯塔附近后，无人机读取实时船体停机坪坐标。
8. 无人机平滑下降到停机坪。
9. 接近触地并稳定后，停机坪约束插件重新锁住无人机。

## 7. 一条命令启动完整演示

推荐直接使用集成 launch：

```bash
source /opt/ros/humble/setup.bash
source $UAV_USV_INSTALL/setup.bash

ros2 launch uav_usv_sim uav_usv_cooperation_demo.launch.py px4_dir:=$PX4_DIR
```

带常用参数的示例：

```bash
ros2 launch uav_usv_sim uav_usv_cooperation_demo.launch.py \
  px4_dir:=$PX4_DIR \
  target_radius:=5.0 \
  drone_depart_delay:=8.0 \
  takeoff_climb_rate:=1.0 \
  drone_cruise_speed:=2.5 \
  drone_deck_approach_speed:=1.2 \
  deck_land_altitude:=0.1 \
  deck_descent_rate:=0.3 \
  deck_touchdown_tolerance:=0.12 \
  deck_touchdown_hold_time:=0.8
```

## 8. 常用参数解释

`target_x`、`target_y`

灯塔目标点坐标，默认是 Gazebo 世界中的 `(35.0, 18.0)`。

`target_radius`

灯塔周围的固定判定半径，默认 `5.0` 米。船进入这个半径后认为到达灯塔附近；无人机飞向这个半径圆上的安全点，不直接飞向灯塔中心。

`takeoff_altitude`

无人机起飞后的目标高度，默认 `8.0` 米。

`takeoff_climb_rate`

起飞爬升速度，数值越大起飞越快，默认 `0.8` m/s。

`drone_depart_delay`

无人船开始向灯塔移动后，无人机等待多久再飞向灯塔，默认 `10.0` 秒。

`drone_cruise_speed`

无人机飞向灯塔安全点时的速度限制。

`drone_deck_approach_speed`

无人机靠近船体停机坪时的速度限制。

`deck_hover_altitude`

降落前，无人机在停机坪上方悬停的高度。

`deck_align_radius`

无人机进入停机坪上方多大范围后，开始短暂停稳再下降。

`deck_settle_time`

无人机对准停机坪后，下降前停稳的时间。

`deck_descent_rate`

最后下降阶段的垂直速度。

`deck_land_altitude`

最终认为落到甲板附近的高度偏移。

`deck_touchdown_tolerance`

重新锁定停机坪前，Gazebo 实际位置允许的水平和垂直误差。默认 `0.12` 米，用来避免最后看起来像被停机坪吸过去。

`deck_touchdown_hold_time`

无人机到达触地点后保持多久再重新锁定甲板。

## 9. MAVLink 与坐标关系说明

协同节点使用 `pymavlink` 向 PX4 发送 MAVLink setpoint。

PX4 本地坐标使用 NED：

- `x` 指向北/前向
- `y` 指向东/右向
- `z` 向下，所以上升高度对应负的 `z`

Gazebo 世界坐标是常见 ENU/世界坐标形式：

- `x`、`y` 是水平平面坐标
- `z` 向上

PX4 Gazebo bridge 中存在坐标转换。本节点会根据 Gazebo 中无人机实时位姿和 PX4 `LOCAL_POSITION_NED` 自动标定二者关系，然后在控制时进行转换。因此任务参数里的灯塔、船体和停机坪位置都可以按 Gazebo 世界坐标理解。

## 10. 常见问题

### PX4 没启动或 MAVLink 连不上

确认 PX4 路径存在：

```bash
ls $PX4_DIR
```

确认启动 PX4 + Gazebo：

```bash
ros2 launch uav_usv_sim uav_usv_px4_sim.launch.py px4_dir:=$PX4_DIR
```

协同节点默认连接：

```text
udp:127.0.0.1:14540
```

### 找不到模型或插件

重新编译并同步：

```bash
cd $UAV_USV_WS
source /opt/ros/humble/setup.bash

colcon --log-base $UAV_USV_LOG build \
  --build-base $UAV_USV_BUILD \
  --install-base $UAV_USV_INSTALL \
  --symlink-install

source $UAV_USV_INSTALL/setup.bash
ros2 run uav_usv_sim sync_to_px4.sh
```

### /home 空间不够

使用 `/var/tmp` 构建路径，不要默认在工作区下生成大量 build/install/log：

```bash
colcon --log-base $UAV_USV_LOG build \
  --build-base $UAV_USV_BUILD \
  --install-base $UAV_USV_INSTALL \
  --symlink-install
```

### 关闭残留仿真进程

如果上次仿真异常退出，可以检查：

```bash
pgrep -af 'px4|gz sim|ruby.*gz|cooperative_lighthouse|make px4_sitl|ros2 launch uav_usv'
```

必要时手动结束相关进程。

## 11. 主要文件说明

- `worlds/default.sdf`：Gazebo 世界，包含海面、灯塔、无人船和停机坪约束插件。
- `models/simple_boat`：无人船模型和停机坪。
- `models/waves`：海浪模型。
- `plugins/BoatWaveFollower.cc`：让船随波浪运动的 Gazebo 插件。
- `plugins/DroneDeckFollower.cc`：无人机停在甲板时的跟随/锁定插件。
- `scripts/cooperative_lighthouse_mission.py`：协同任务主节点，负责 MAVLink 控制无人机和 Gazebo Transport 控制无人船。
- `launch/uav_usv_px4_sim.launch.py`：启动 PX4 + Gazebo。
- `launch/cooperative_lighthouse_mission.launch.py`：只启动协同任务节点。
- `launch/uav_usv_cooperation_demo.launch.py`：一条命令启动完整协同演示。
